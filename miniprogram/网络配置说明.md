# 真机测试网络配置指南

## 📋 问题说明

微信小程序在**真机测试**时无法访问 `localhost` 或 `127.0.0.1`，因为手机无法访问电脑的本地回环地址。

---

## 🔧 解决方案

### 方案一：使用局域网 IP（推荐）

#### 1. 查看电脑 IP 地址

**Windows 系统**：
```bash
# 命令提示符执行
ipconfig

# 找到"无线局域网适配器 WLAN"或"以太网适配器"
# IPv4 地址 就是电脑的局域网 IP
# 例如：192.168.1.100 或 10.14.168.76
```

**常见 IP 段**：
- `192.168.x.x`（最常见）
- `10.x.x.x`
- `172.16.x.x` - `172.31.x.x`

#### 2. 修改小程序配置

编辑 `app.js`：
```javascript
globalData: {
    // ❌ 错误：真机无法访问
    baseUrl: 'http://localhost:8000/api',
    
    // ✅ 正确：使用电脑 IP
    baseUrl: 'http://192.168.1.100:8000/api'
}
```

#### 3. 确保手机和电脑在同一 WiFi

- 电脑和手机必须连接**同一个 WiFi 网络**
- 不要用手机的 4G/5G 流量

#### 4. 关闭防火墙或添加规则

**Windows 防火墙**：
1. 控制面板 → Windows Defender 防火墙
2. 高级设置 → 入站规则 → 新建规则
3. 端口 → TCP → 特定本地端口 → `8000`
4. 允许连接 → 完成

**临时关闭防火墙（测试用）**：
1. 控制面板 → Windows Defender 防火墙
2. 启用或关闭防火墙
3. 关闭专用网络防火墙

---

### 方案二：使用内网穿透工具（适合远程测试）

如果手机和电脑不在同一网络，可以使用内网穿透工具：

#### Ngrok

```bash
# 1. 下载 ngrok
# https://ngrok.com/

# 2. 启动 ngrok
ngrok http 8000

# 3. 获取 https 地址
# 例如：https://abc123.ngrok.io

# 4. 修改小程序配置
baseUrl: 'https://abc123.ngrok.io/api'
```

#### 花生壳

```bash
# 1. 下载花生壳
# https://hsk.oray.com/

# 2. 注册并登录

# 3. 添加内网穿透规则
# 内网地址：localhost:8000

# 4. 获取外网访问地址
# 例如：http://xxx.vicp.net:8000
```

---

### 方案三：使用微信开发者工具模拟器（开发阶段推荐）

开发阶段建议优先使用**模拟器**：

1. 微信开发者工具 → 编译
2. 在模拟器中测试
3. 不需要修改 IP 配置，使用 `localhost` 即可

**模拟器优点**：
- ✅ 调试方便（Console、Network 面板）
- ✅ 不需要配置网络
- ✅ 开发效率高

**真机测试时机**：
- 功能开发完成后
- 需要测试真实用户体验
- 需要测试摄像头、GPS 等硬件功能

## ⚠️ 常见问题

### Q1: 真机测试提示"网络错误"？

**A**: 
1. 检查是否使用了电脑 IP 而非 localhost
2. 确认手机和电脑在同一 WiFi
3. 检查防火墙设置
4. 确认后端服务正常运行

### Q2: IP 地址总是变化怎么办？

**A**: 
1. 在路由器中设置静态 IP
2. 每次变化后手动更新 `app.js`
3. 使用内网穿透工具（有固定域名）

### Q3: 真机测试需要配置 HTTPS 吗？

**A**: 
- **开发阶段**：在微信开发者工具中勾选"不校验合法域名"
- **生产环境**：必须使用 HTTPS 域名

### Q4: 模拟器能访问，真机不能访问？

**A**: 
这是正常的，因为：
- 模拟器运行在电脑上，可以访问 localhost
- 真机是独立设备，无法访问电脑的 localhost

## 🔄 IP 变化时的更新步骤

如果电脑 IP 变化（如重启路由器后）：

1. 查看新 IP：
   ```bash
   ipconfig
   ```

2. 修改 `app.js`：
   ```javascript
   baseUrl: 'http://新 IP:8000/api'
   ```

3. 重新编译小程序
