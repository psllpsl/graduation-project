## 🐛 问题与解决方案

### 问题 1：图标文件缺失

**现象**：小程序启动报错，找不到图标文件

**解决**：
- 创建图标生成脚本（generate_icons.py）
- 生成 10 个占位 PNG 文件
- 后续可替换为正式图标

### 问题 2：微信登录失败

**现象**：点击登录提示"网络请求失败"

**原因**：
1. 后端缺少 wx-login 接口
2. AppID 不合法（使用测试号）
3. 未配置"不校验合法域名"

**解决**：
- 后端添加 wx-login 接口（测试模式）
- 使用微信测试账号
- 勾选"不校验合法域名"

### 问题 3：路由冲突

**现象**：/api/patients/openid/{openid} 返回 404

**原因**：FastAPI 路由顺序问题，/{patient_id} 先匹配了

**解决**：
- 修改路由为 /by-openid/{openid}
- 确保固定前缀路由在动态参数之前

### 问题 4：会话历史 401 错误

**现象**：加载会话历史返回未授权

**原因**：get_current_user 期望 sub 是整数，但患者 Token 是 "patient:25" 格式

**解决**：
- 使用 HTTPBearer 直接解析 Token
- 自定义 get_patient_id_from_token 函数

### 问题 5：搜索接口 401

**现象**：知识库搜索返回 401，登录状态丢失

**原因**：
1. 小程序 API 路径与后端不匹配
2. 401 处理逻辑清除 Token

**解决**：
- 修改 API 路径：/search → /search/query
- 优化 401 处理：只在有 Token 时才清除

### 问题 6：切换用户聊天记录不清除

**现象**：退出登录后重新登录，聊天记录还在

**解决**：
- 登录时清除旧 sessionId
- onShow 中检查 sessionId 变化并清空消息

### 问题 7：快捷问题无法发送

**现象**：点击快捷问题提示"无法发送 canSend=false"

**原因**：没有设置 canSend 为 true

**解决**：onQuickQuestion 中同时设置 inputValue 和 canSend

### 问题 8：输入框残留文本

**现象**：回车发送后输入框还有文本

**解决**：发送时立即清空输入框

### 问题 9：AI 回复被截断

**现象**：AI 回复不完整，后续对话出问题

**解决**：
- 提示词：150-200 字 → 简单 100-150 字，复杂 200-300 字
- 后处理截断：250 字 → 500 字
- max_tokens：150 → 500

### 问题 10：真机测试网络错误

**现象**：真机提示网络错误

**原因**：使用 localhost，手机无法访问

**解决**：
- 使用电脑局域网 IP（10.14.168.76）
- 后端监听 0.0.0.0
- 手机和电脑同一 WiFi
