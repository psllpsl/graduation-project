# AutoDL 服务配置快速更新

## 📝 你的 AutoDL 服务地址格式

根据你的实例，服务地址格式为：

```
https://你的实例 ID.seetacloud.com:端口
```

---

## 🔧 更新后端配置

### 方式一：直接编辑 `.env` 文件

在本地后端项目中，编辑 `backend/.env`：

```env
# AI 服务配置
AI_SERVICE_URL=https://你的实例 ID.seetacloud.com:端口/generate
AI_SERVICE_TYPE=autodl
AI_MAX_TOKENS=150
AI_TEMPERATURE=0.7
AI_TIMEOUT_SECONDS=60
```

**关键**：确保 `AI_SERVICE_URL` 末尾有 `/generate`

---

### 方式二：使用 PowerShell 快速更新

在本地电脑，后端项目目录下执行：

```powershell
cd D:\Project\毕业设计\backend

# 备份原配置
Copy-Item .env .env.bak

# 更新 AI_SERVICE_URL
(Get-Content .env) -replace '^AI_SERVICE_URL=.*', 'AI_SERVICE_URL=https://你的实例 ID.seetacloud.com:端口/generate' | Set-Content .env

# 验证更新
Select-String "AI_SERVICE_URL" .env
```

---

### 方式三：使用批处理更新

创建 `update_config.bat`：

```batch
@echo off
chcp 65001 >nul
cd /d "%~dp0backend"

echo 更新 AI 服务配置...
powershell -Command "(Get-Content '.env') -replace '^AI_SERVICE_URL=.*', 'AI_SERVICE_URL=https://你的实例 ID.seetacloud.com:端口/generate' | Set-Content '.env'"

echo 配置已更新
Select-String "AI_SERVICE_URL" .env
pause
```

## 🔄 如果更换了 AutoDL 实例

1. 在 AutoDL 上运行启动脚本：`bash start.sh`
2. 输入 `y` 表示更换实例
3. 输入新的公网地址：`https://新实例地址.seetacloud.com:端口`
4. 脚本会自动保存配置
5. 在本地更新 `backend/.env` 中的 `AI_SERVICE_URL`

---

## ⚠️ 注意事项

1. **URL 格式**：必须是 `https://` 开头
2. **端口号**：通常是 `8443`（不是 6008）
3. **路径**：末尾必须有 `/generate`
4. **不要有空格**：URL 中不能有空格

## 🆘 故障排查

### 问题 1：连接超时

**检查**：
- AutoDL 服务是否运行：在 AutoDL 控制台查看实例状态
- 自定义服务是否添加：端口 8443 是否映射
- URL 是否正确：复制完整的公网地址

### 问题 2：SSL 证书错误

**解决**：在 `ai_service.py` 中添加 `verify=False`

```python
with httpx.Client(timeout=self.timeout, verify=False) as client:
    response = client.post(...)
```

### 问题 3：404 错误

**检查**：URL 是否完整，应该是 `.../generate` 结尾
