# 后端服务启动说明

## 🚀 快速启动（推荐）

### 方式一：使用启动脚本（交互式）

**双击运行**：
```
start.bat
```

**或命令行运行**：
```bash
cd D:\Project\毕业设计\backend
python start_server.py
```

**启动流程**：
1. 脚本会显示当前 AI 服务地址
2. 输入新的 AI 服务地址（来自 AutoDL 控制台）
3. 自动更新 `.env` 配置文件
4. 测试 AI 服务连接
5. 启动 FastAPI 后端服务

登录 AutoDL 控制台，找到你的实例：

- **外网端口**：`8443`
- **完整地址**：`https://uu769760-8aa7-e51ad73a.bjb1.seetacloud.com:8443`

**注意**：实例重启后 URL 会变化，端口永远是 8443（外网）/ 6008（内网）

---

### 2. 启动后端服务

#### 方法 A：使用交互式启动脚本（推荐）

```bash
# 方式 1：双击 start.bat
# 方式 2：命令行运行
python start_server.py
```

脚本会自动：
- 提示你输入 AI 服务地址
- 更新配置文件
- 测试连接
- 启动服务

#### 方法 B：手动启动（老手适用）

```bash
# 1. 编辑 .env 文件，更新 AI_SERVICE_URL
# 2. 激活虚拟环境
venv\Scripts\activate

# 3. 启动服务
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

## 🆘 常见问题

### Q1: AI 服务地址变了怎么办？

**A**: 重新运行 `start.bat` 或 `python start_server.py`，脚本会提示你输入新地址。

### Q2: 忘记 AI 服务地址了？

**A**: 
1. 登录 AutoDL 控制台
2. 找到运行中的实例
3. 查看"自定义服务"或"外网端口"
4. 复制完整 URL

### Q3: 后端启动失败？

**A**: 检查：
- Python 虚拟环境是否激活
- 依赖是否安装：`pip install -r requirements.txt`
- 端口 8000 是否被占用
- 数据库是否运行

### Q4: AI 对话返回错误？

**A**: 
1. 查看后端日志中的 AI 服务调用信息
2. 确认 AI 服务地址正确
3. 在 AutoDL 终端测试本地调用
4. 检查网络连接
