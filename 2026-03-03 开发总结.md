## 📝 详细开发内容

### 1. 小程序基础框架

**文件结构**：
```
miniprogram/
├── app.js, app.json, app.wxss
├── pages/
│   ├── index/ (对话页面)
│   ├── appointment/ (复诊页面)
│   ├── knowledge/ (知识页面)
│   └── profile/ (个人中心)
├── utils/
│   ├── api.js (网络请求封装)
│   └── util.js (工具函数)
└── images/ (图标资源)
```

**核心功能**：
- TabBar 导航（4 个 tab）
- 全局样式与主题色
- 网络请求封装（自动携带 Token）
- 401 自动处理

### 2. 用户认证

**流程**：
```
用户点击登录
  ↓
wx.login() 获取 code
  ↓
发送到后端 /api/auth/wx-login
  ↓
后端返回 Token + 患者信息
  ↓
保存 Token 到本地存储
  ↓
后续请求自动携带 Token
```

**关键代码**：
- `app.js` - wxLogin() 方法
- `backend/app/api/auth.py` - wx-login 接口

### 3. 对话功能

**UI 组件**：
- 消息列表（scroll-view）
- 消息气泡（用户/AI 样式）
- 输入框与发送按钮
- 快捷问题列表
- 加载动画

**功能特性**：
- 自动滚动到底部
- 输入框固定底部
- 会话历史加载
- 切换用户自动清空

**API**：
- `POST /api/dialogues/chat` - 发送消息
- `GET /api/dialogues/session/{sessionId}` - 加载历史

### 4. 复诊计划

**功能**：
- 复诊列表展示
- 状态统计（待复诊/已完成/总计）
- 复诊详情查看
- 确认/取消复诊

**API**：
- `GET /api/appointments/patient/my` - 获取我的复诊
- `PATCH /api/appointments/{id}/status` - 更新状态

### 5. 知识库

**功能**：
- 知识列表（分页）
- 分类筛选
- 搜索功能
- 知识详情弹窗

**API**：
- `GET /api/knowledge/` - 获取列表
- `GET /api/knowledge/categories` - 获取分类
- `GET /api/knowledge/search/query` - 搜索

### 6. 后端接口优化

**新增接口**：
- `POST /api/auth/wx-login` - 微信登录
- `GET /api/patients/by-openid/{openid}` - 按 openid 查询
- `POST /api/dialogues/chat` - 患者对话
- `GET /api/appointments/patient/my` - 我的复诊

**认证优化**：
- 使用 HTTPBearer 解析患者 Token
- 从 Token 提取 patient_id
- 公开知识库和复诊查询接口

### 7. AI 服务优化

**提示词优化**：
- 名字：小齿
- 身份：牙科修复术后护理专家
- 输出长度：简单 100-150 字，复杂 200-300 字
- 风格：亲切友好、专业可靠

**参数调整**：
- max_tokens: 150 → 500
- 后处理截断：250 字 → 500 字

### 8. 真机测试

**配置步骤**：
1. 查看电脑 IP：`ipconfig` → `10.14.168.76`
2. 修改 `app.js` 中的 `baseUrl`
3. 后端监听 `0.0.0.0`
4. 微信开发者工具勾选"不校验合法域名"

**网络要求**：
- 手机和电脑同一 WiFi
- 防火墙允许 8000 端口

## 🐛 问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 图标缺失 | 文件未创建 | 生成占位 PNG |
| 登录失败 | 接口缺失、AppID 问题 | 添加接口、使用测试号 |
| 路由 404 | 路由顺序问题 | 修改路由名称 |
| 会话 401 | Token 解析失败 | 使用 HTTPBearer |
| 搜索 401 | API 路径不匹配 | 修改 API 路径 |
| 切换用户不清空 | sessionId 未清除 | 登录时清除旧 sessionId |
| 快捷问题无法发送 | canSend 未设置 | 同时设置 inputValue 和 canSend |
| 输入框残留 | 清空时机不对 | 发送时立即清空 |
| AI 回复截断 | 限制太严 | 放宽到 500 字 |
| 真机网络错误 | 使用 localhost | 使用局域网 IP |
