## 三、数据库搭建

### 3.1 创建数据库

```bash
# 登录 MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE dental_clinic CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 创建用户（可选）
CREATE USER 'dental_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON dental_clinic.* TO 'dental_user'@'localhost';
FLUSH PRIVILEGES;

# 使用数据库
USE dental_clinic;
```

### 3.2 导入表结构

```bash
# 进入文档目录
cd D:\Project\毕业设计\docs\数据库设计

# 执行建表脚本
mysql -u root -p dental_clinic < create_tables.sql
```

### 3.3 导入初始化数据

```bash
# 导入测试数据
mysql -u root -p dental_clinic < init_data.sql
```

### 3.4 验证数据库

```bash
# 查看表
mysql -u root -p dental_clinic -e "SHOW TABLES;"

# 应看到 7 张表：
# - users
# - patients
# - treatment_records
# - appointments
# - dialogues
# - knowledge_base
# - system_config
```

## 四、后端服务部署

### 4.1 进入后端目录

```bash
cd D:\Project\毕业设计\backend
```

### 4.2 创建虚拟环境

```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境（Windows）
venv\Scripts\activate
```

### 4.3 安装依赖

```bash
pip install -r requirements.txt
```

> 💡 **提示**：如果下载慢，使用国内镜像：
> ```bash
> pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
> ```

### 4.4 配置环境变量

编辑 `backend/app/config.py` 或直接使用 `.env` 文件：

```python
# 数据库配置
DATABASE_URL = "mysql+pymysql://root:你的密码@localhost:3306/dental_clinic"

# JWT 配置
SECRET_KEY = "your-secret-key-change-in-production"

# AI 服务配置
AI_SERVICE_URL = "https://你的实例 ID.seetacloud.com:端口/generate"
AI_SERVICE_TYPE = "autodl"
AI_MAX_TOKENS = 500
AI_TEMPERATURE = 0.7
AI_TIMEOUT_SECONDS = 60
```

### 4.5 启动后端服务

**方式一：使用启动脚本（推荐）**

```bash
# Windows
start_server.bat

# 或双击
backend\start_server.bat
```

**方式二：手动启动**

```bash
# 开发模式（自动重载）
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 生产模式
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
```

### 4.6 验证后端服务

**访问 API 文档**：
- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

**测试登录接口**：

1. 打开 http://localhost:8000/docs
2. 调用 `POST /api/auth/login`
3. 输入测试账号：
   - 用户名：`admin`
   - 密码：`admin123`
4. 成功应返回 `access_token`

## 八、系统使用指南

### 8.1 患者使用流程（微信小程序）

#### 步骤 1：登录/注册

1. 打开微信小程序
2. 微信授权一键登录
3. 自动绑定患者档案（如已存在）

#### 步骤 2：AI 对话咨询

1. 进入首页（对话界面）
2. 输入问题或选择快捷问题
3. 获取 AI 专业回复

#### 步骤 3：查看复诊计划

1. 点击底部 "复诊" 标签
2. 查看个人复诊时间表
3. 确认或取消复诊

#### 步骤 4：浏览知识库

1. 点击底部 "知识" 标签
2. 浏览术后护理知识
3. 搜索感兴趣的内容

### 8.2 医护人员使用流程（Streamlit 后台）

#### 步骤 1：登录系统

1. 访问 http://localhost:8501
2. 输入账号密码
3. 点击登录

#### 步骤 2：查看仪表盘

1. 查看系统概览数据
2. 监控复诊趋势、对话量
3. 查看患者分布统计

#### 步骤 3：患者管理

1. 进入 "患者管理" 页面
2. 点击 "新增患者" 创建档案
3. 编辑/删除患者信息
4. 查看患者详情

#### 步骤 4：复诊管理

1. 进入 "复诊管理" 页面
2. 点击 "创建复诊计划"
3. 选择患者、日期、治疗类型
4. 更新复诊状态

#### 步骤 5：对话监管

1. 进入 "对话监管" 页面
2. 查看 AI 对话记录
3. 必要时进行人工接管

#### 步骤 6：知识库管理

1. 进入 "知识库管理" 页面
2. 新增/编辑/删除知识条目
3. 按分类管理知识

#### 步骤 7：用户管理

**注册新用户**：
1. 登录页 → 点击 "📝 注册账号"
2. 填写新用户信息
3. 输入管理员账号和密码验证
4. 提交注册

**重置密码**：
1. 登录页 → 点击 "🔑 忘记密码"
2. 输入需要重置的用户名
3. 输入管理员账号和密码验证
4. 输入新密码并确认
5. 提交重置

## 九、常见问题

### 9.1 后端服务问题

#### Q1: 启动失败 "Address already in use"

**原因**：8000 端口被占用

**解决方案**：
```bash
# 运行停止脚本
stop_all.bat

# 或手动查找并停止进程
netstat -ano | findstr :8000
taskkill /F /PID <进程 ID>
```

#### Q2: 数据库连接失败

**检查项**：
1. MySQL 服务是否启动
2. 数据库名、用户名、密码是否正确
3. 网络连接是否正常

**解决方案**：
```bash
# 启动 MySQL（Windows 服务）
net start MySQL80

# 或使用 Docker
docker start mysql-dental
```

#### Q3: AI 服务调用失败

**检查项**：
1. AI_SERVICE_URL 是否正确
2. 网络是否通畅
3. AutoDL 实例是否运行

**解决方案**：
- 检查 AutoDL 实例状态
- 重新获取实例 URL
- 测试网络连接

### 9.2 小程序问题

#### Q1: 网络请求失败

**原因**：域名未配置或 IP 错误

**解决方案**：
1. 修改 `app.js` 中的 `baseUrl` 为正确的电脑 IP
2. 确保手机和电脑在同一 WiFi
3. 勾选 "不校验合法域名"

#### Q2: 登录失败

**检查项**：
1. 后端服务是否启动
2. AppID 是否正确
3. 患者是否存在

**解决方案**：
- 检查后端日志
- 确认患者档案已创建

### 9.3 Streamlit 后台问题

#### Q1: 登录失败 "API 请求失败"

**原因**：后端服务未启动

**解决方案**：
```bash
# 启动后端服务
cd backend
start_server.bat
```

#### Q2: 端口已被占用

**解决方案**：
```bash
# 使用其他端口
streamlit run app.py --server.port 8502
```

#### Q3: 图表不显示

**解决方案**：
```bash
# 重新安装 plotly
pip install --upgrade plotly
```

### 9.4 数据库问题

#### Q1: 表不存在

**解决方案**：
```bash
# 重新导入建表脚本
mysql -u root -p dental_clinic < docs/数据库设计/create_tables.sql
```

#### Q2: 数据为空

**解决方案**：
```bash
# 导入初始化数据
mysql -u root -p dental_clinic < docs/数据库设计/init_data.sql
```
